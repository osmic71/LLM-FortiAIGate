# Importing a Custom Log Parser into FortiAnalyzer

## Overview

FortiAnalyzer supports custom log parsers that extend its ability to ingest and normalize log data from non-standard or custom log formats. The **FortiAIGate Log Parser** is a custom parser designed to handle the proprietary `FSFADOM3-SYS` glog format generated by FortiAIGate appliances, enabling full log visibility, field normalization, and reporting within FortiAnalyzer.

> **Parser file:** [`FortiAIGate Log Parser.json`](https://github.com/osmic71/LLM-FortiAIGate/blob/main/FortiAIGate%20Log%20Parser.json)  

---

## Prerequisites

| Requirement | Details |
|---|---|
| FortiAnalyzer version | 7.4.x or later (recommended) |
| Admin role | `super_admin` or equivalent with Log Parser management permissions |
| Parser file | `FortiAIGate Log Parser.json` (downloaded from the repository above) |
| Log source | FortiAIGate device registered and forwarding logs to FortiAnalyzer |

---

## Step 1 — Download the Parser File

Download the parser JSON from the GitHub repository:

```bash
curl -L -o "FortiAIGate Log Parser.json" \
  "https://raw.githubusercontent.com/osmic71/LLM-FortiAIGate/main/FortiAIGate%20Log%20Parser.json"
```

Or download it manually from the [GitHub repository](https://github.com/osmic71/LLM-FortiAIGate/blob/main/FortiAIGate%20Log%20Parser.json) using the **Download raw file** button.

---

## Step 2 — Access the Log Parser Section in FortiAnalyzer

1. Log in to your **FortiAnalyzer** web UI as an administrator.
2. Navigate to:

   ```
   Incidents & Alerts → Log Parser
   ```

3. The **Log Parser** page displays all currently installed parsers (built-in and custom).

---

## Step 3 — Import the Parser

1. Click **More**
2. Click **Import** (top-right of the Log Parser list).
3. In the import dialog, click **Browse** and select the downloaded `FortiAIGate Log Parser.json` file.
4. Click **OK** / **Upload** to proceed.
5. FortiAnalyzer will validate the JSON schema. If the file is valid, the parser will appear in the list with status **Enabled**.



---

## Step 4 — Verify the Parser

After import, confirm the parser is correctly loaded:

1. In the **Log Parser** list, locate `FortiAIGate Log Parser`.
2. Click the parser name to inspect its **Field Definitions** and **Match Rules**.
3. Confirm the following key fields are present:

   | Field Name | Type | Description |
   |---|---|---|
   | `date_time` | datetime | Timestamp of the log event |
   | `data_sourcetype` | string | Log source identifier (value: `FortiAIGate`) |
   | `event_profile` | string | AI Flow name (e.g., `flow_fortiaigate`) |
   | `event_policy` | string | AI Guard policy matched (e.g., `default_ollama`) |
   | `event_type` | string | Log event type (e.g., `TRAFFIC`) |
   | `srcip` | ip | Source IP address of the request |
   | `event_message` | string | Full prompt or message content sent to the LLM |
   | `event_tags` | string | Tags associated with the event |
   | `http_response_body` | string | Response body returned by the LLM |
   | `threat_type` | string | Threat classification detected (e.g., prompt injection, DLP) |
   | `event_status` | string | Processing status of the event |
   | `event_action` | string | Action taken by the policy (e.g., `allow`, `block`) |
   | `event_source` | string | Origin source of the log event |
   | `application_service` | string | Backend AI service targeted |
   | `event_count` | integer | Number of aggregated events |

---

## Step 5 — Associate the Parser with a Log Device

Associating a parser to a device is a **two-part process**: the device must first be approved in Device Manager, and then the parser assignment is configured from the Log Parser section.

### Part A — Approve the Device in Device Manager

1. Navigate to **Device Manager**.
2. Locate the FortiAIGate syslog source (e.g., `SYSLOG-<device-id>`) under **Unauthorized Devices** or the pending device list.
3. Right-click the device and select **Authorize** (or click **Add** and confirm the device details).
4. Once authorized, the device will appear in the managed device list and begin forwarding logs to FortiAnalyzer.

> ⚠️ The parser assignment will not be available until the device has been authorized in Device Manager.

### Part B — Assign the Parser from Log Parser

1. Navigate back to:
   ```
   Incident & Alerts → Log Parser
   ```
2. Scroll down to the **Assigned Parser** section (below the parser list).
3. Click **Create New** to add a new parser assignment.
4. In the assignment dialog, configure the following:

   | Field | Value |
   |---|---|
   | **Device ID** | Select the authorized device (e.g., `SYSLOG-ACD4218E`) |
   | **Application** | `Syslog` |
   | **Parser** | `FortiAIGate Log Parser` |

5. Click **OK** to save the assignment.

The selected device will now use the **FortiAIGate Log Parser** to normalize all incoming syslog data.

---
## Step 6 — Validate Log Ingestion and Create a Custom View

### Part A — Verify Log Parsing

1. Navigate to **Log View** and select the ADOM where the FortiAIGate device was authorized.
2. In the device tree on the left panel, select the syslog source (e.g., `SYSLOG-ACD4218E`).
3. In the filter bar (Text Mode), apply the following filter to scope logs to FortiAIGate:

   ```
   devname="FortiAIGate"
   ```

4. Confirm that logs are displayed and that fields such as `Event Profile`, `Event Policy`, `Event Type`, `Source IP`, and `Event Message` are populating correctly.

> ⚠️ If no logs appear or fields are empty, revisit the parser assignment in Step 5 and confirm the device is forwarding logs in the expected format.

---

### Part B — Create a Custom View

A Custom View saves the filter and column layout as a persistent view for ongoing FortiAIGate AI traffic monitoring.

1. In **Log View**, click **Custom View** in the top toolbar, then select **New Custom View**.
2. Name the view:

   ```
   FortiAIGate - AI Traffic
   ```

3. Set the filter to:

   ```
   data_sourcetype="FortiAIGate"
   ```

4. Click the **Column Settings** icon (gear) and enable the following columns in order:

   | # | Column Label | Field Key |
   |---|---|---|
   | 1 | Date/Time | `date_time` |
   | 2 | Data Source Type | `data_sourcetype` |
   | 3 | Event Profile | `event_profile` |
   | 4 | Event Policy | `event_policy` |
   | 5 | Event Type | `event_type` |
   | 6 | Source IP | `srcip` |
   | 7 | Event Message | `event_message` |
   | 8 | Event Tags | `event_tags` |
   | 9 | HTTP Response Body | `http_response_body` |
   | 10 | Threat Type | `threat_type` |
   | 11 | Event Status | `event_status` |
   | 12 | Event Action | `event_action` |
   | 13 | Event Source | `event_source` |
   | 14 | Application Service | `application_service` |
   | 15 | Event Count | `event_count` |

5. Arrange columns in the order above using drag-and-drop.
6. Click **Save**. The view will appear in the left panel under **Log View** for quick access.
